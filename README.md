# 讲题评测平台

用于评测“小学数学 AI 讲题能力”的双模型对话平台。

平台由两个模型自动完成一轮讲题对话：

1. **AI 老师模型**负责理解题目、分步讲解，并根据学生反馈继续教学。
2. **学生模型**模拟真实小学生，对讲解进行追问、打断、反馈，或要求重讲和更换方法。

对话结束后，可以按照统一标准进行人工评分，也可以调用 Judge 模型生成参考评分，并导出本轮评测的完整数据。

## 在线使用

### [打开讲题评测平台](https://math-dialogue-evaluator.netlify.app)

直接通过浏览器访问即可，无需安装软件。

## 评测流程

```text
输入题目与参考信息
↓
配置学生模型、AI 老师模型和 Judge 模型
↓
启动自动对话
↓
查看完整讲题过程
↓
人工评分或调用 Judge 自动评分
↓
导出 JSON / CSV 数据
```

## 题目输入

平台支持在线填写，也支持一键上传 `.xlsx` Excel 表格批量导入。

Excel 第一行使用以下列名：

| 序号 | 题目 | 原图 | 答案 | 知识点 | 解析 | 错因分析 |
|---|---|---|---|---|---|---|
| 1 | 需要讲解的题目 | 图片 URL 或 data URL | 标准答案 | 涉及的知识点 | 标准解题过程 | 常见错误及原因 |

允许部分单元格为空。系统也兼容“编号、题号、ID、题干、题目文本、图片、题图、题目图片、图片链接、参考答案、标准答案、考点、答案解析、解题过程、错因、错误原因”等常见列名。

“原图”列读取的是单元格里的图片链接或 data URL；如果图片是作为 Excel 内嵌对象插入，当前网页端轻量导入不会解析图片对象本身。在线填写时也可以直接上传本地图片并在页面中预览。

如果图片在本地文件夹中，可以先上传 Excel，再点击“批量上传图片”。平台会按图片文件名中的序号匹配 Excel 行，例如：

- `1.png` 匹配序号 `1`
- `题目_02.jpg` 匹配序号 `2`
- `003.jpeg` 匹配序号 `3`

导入后可以通过题目选择器切换不同题目，并继续在页面中补充或修改内容。

平台中的题目信息包括：

- **题目**：需要讲解的小学数学题
- **序号**：用于批量导入和图片匹配
- **原图**：题目图片，可通过链接、data URL 或在线上传补充
- **参考答案**：题目的标准答案
- **知识点**：题目涉及的数学知识
- **参考解析**：标准解题过程
- **错因分析**：学生常见错误及其原因

除题目外，其余内容均可按需要填写。参考信息会提供给模型用于保证讲解和评分的准确性。

## 双模型对话

### AI 老师模型

AI 老师负责：

- 理解题意并识别考点
- 分步骤讲解，不只给出答案
- 使用适合小学生理解的表达
- 回答学生追问并处理打断
- 根据学生反馈重讲、换方法或降低难度
- 回答追问后回到原来的讲解位置

### 学生模型

学生模型会模拟真实小学生：

- 表达没有听懂
- 追问某一步为什么这样做
- 要求老师重新讲解
- 要求更换解题方法
- 打断老师或反馈当前理解程度
- 理解后明确表示“我懂了”

学生模型不会代替老师完整解题或直接给出答案。

### 对话结束规则

- AI 老师首先发言，学生随后进行反馈
- 两个模型交替生成消息
- 支持自动运行、逐条继续、停止和重置
- 每轮对话不超过 20 条消息
- 学生提前听懂后，老师仍会给出完整总结
- 如果对话持续较长，老师会在消息上限前补齐解题步骤、算式和最终答案

## 评分标准

平台采用 100 分制，综合得分基线为 80 分。

| 一级维度 | 分值 | 二级维度 |
|---|---:|---|
| 题意理解与数学正确性 | 30 | 题意理解、考点识别、解法正确性、答案正确性 |
| 讲题内容质量 | 25 | 步骤完整性、逻辑清晰度、小学生可理解性、易错点提醒、讲解简洁度 |
| 交互问答能力 | 20 | 追问理解、上下文保持、纠错与澄清、情绪支持 |
| 流程控制与多轮稳定性 | 15 | 打断响应、位置恢复、控制指令执行 |
| 语音表现与儿童友好性 | 10 | 发音准确、语速节奏、语气自然、重点突出、切换平滑 |

每个二级维度按 0–5 分评价：

| 分数 | 含义 |
|---:|---|
| 0 | 异常或完全缺失 |
| 1–2 | 存在严重问题 |
| 3 | 存在普通问题 |
| 4 | 表现较好 |
| 5 | 表现优秀 |

第一版没有真实语音，“语音表现与儿童友好性”按照文本朗读后的预期效果进行评价。

## 自动评分

Judge 模型会根据题目、参考信息、完整师生对话和评分标准生成：

- 各二级维度分数
- 各一级维度小计
- 综合得分和是否达到基线
- 错误标签
- 具体扣分原因
- 改进建议

自动评分仅作为参考，不会直接覆盖人工评分。需要时可以一键将参考分填入人工评分区，再进行人工调整。

如果 Judge 返回的内容不是合法 JSON，平台会保留原始输出并展示解析错误，不会导致页面崩溃。

## 脚本批量评测

如果模型响应很慢，浏览器或部署平台可能会因为等待时间过长而中断请求。此时可以使用仓库内置脚本在本地批量运行评测，绕开浏览器页面的等待限制。

脚本会读取 Excel 题目表，按顺序调用 AI 老师模型、学生模型和 Judge 模型，并把每道题的对话和自动评分写入 JSON 文件。运行过程中会逐题保存结果，中途失败也能保留已经完成的数据。

### 准备 Excel

Excel 第一行仍使用网页端相同的列名：

| 序号 | 题目 | 原图 | 答案 | 知识点 | 解析 | 错因分析 |
|---|---|---|---|---|---|---|

允许部分内容为空。脚本会读取“原图”里的 URL 或 data URL；如果是本地图片文件夹，建议先把图片链接整理进表格，或继续使用网页端的批量图片匹配功能。

### 配置模型

如果三个角色共用同一个模型接口，可以这样配置：

```bash
export MODEL_ENDPOINT="https://ark.cn-beijing.volces.com/api/v3/chat/completions"
export MODEL_API_KEY="你的 API Key"
export MODEL_NAME="你的模型名称或接入点 ID"
```

如果 AI 老师、学生和 Judge 分别使用不同模型，可以分别配置：

```bash
export TEACHER_ENDPOINT="https://ark.cn-beijing.volces.com/api/v3/chat/completions"
export TEACHER_API_KEY="老师模型 API Key"
export TEACHER_MODEL="老师模型名称或接入点 ID"

export STUDENT_ENDPOINT="https://ark.cn-beijing.volces.com/api/v3/chat/completions"
export STUDENT_API_KEY="学生模型 API Key"
export STUDENT_MODEL="学生模型名称或接入点 ID"

export JUDGE_ENDPOINT="https://ark.cn-beijing.volces.com/api/v3/chat/completions"
export JUDGE_API_KEY="Judge 模型 API Key"
export JUDGE_MODEL="Judge 模型名称或接入点 ID"
```

### 开始运行

```bash
npm run evaluate -- --input ./questions.xlsx --output ./outputs/result.json
```

常用参数：

```bash
# 只跑前 3 道题，方便先试接口
npm run evaluate -- --input ./questions.xlsx --limit 3

# 每题最多 20 条消息，默认就是 20
npm run evaluate -- --input ./questions.xlsx --max-messages 20

# 只跑师生对话，不调用 Judge
npm run evaluate -- --input ./questions.xlsx --no-judge
```

输出 JSON 会包含：

- 题目信息
- 完整师生对话
- 每轮使用的模型、Prompt 快照和时间
- Judge 自动评分
- 错误信息

脚本不会把 API Key 写入结果文件。

## 错误标签与备注

人工评测支持多选错误标签，包括：

- 数学答案错误
- 解法错误
- 题意理解错误
- 步骤跳步
- 解释太成人化
- 易错点提醒不足
- 没有回答学生追问
- 上下文丢失
- 打断处理失败
- 流程控制失败
- 语言不适合小学生
- 回答过长
- 重复啰嗦
- 没有鼓励学生

还可以填写人工备注，记录具体问题、优点和复盘结论。

## 模型配置

学生、AI 老师和 Judge 均支持 OpenAI-compatible API，需要分别填写：

- API 地址
- 模型名称或推理接入点 ID
- API Key
- 对应角色的 Prompt

常用 API 地址示例：

```text
OpenAI
https://api.openai.com/v1/chat/completions

火山方舟
https://ark.cn-beijing.volces.com/api/v3/chat/completions
```

`ark-` 开头的 Key 应配合火山方舟地址使用，不能填写 OpenAI 地址。

## 数据导出

支持导出 JSON 和 CSV。导出数据包括：

- 题目、原图、答案、知识点、解析和错因分析
- 三个模型的 API 地址、模型名称和 Prompt 快照
- 每轮对话的角色、内容、模型、轮次和时间
- 人工评分、一级维度小计、总分、错误标签和备注
- 自动评分、扣分原因、错误标签和改进建议
- Session ID、创建时间和导出时间

为了避免泄露凭据，API Key 不会写入导出文件。

## 数据与安全

- 当前评测状态保存在浏览器页面内存中
- 页面刷新后，未导出的本轮数据不会保留
- 模型请求通过平台的服务端接口转发
- API Key 不会写入 GitHub 仓库或导出文件
- 请不要将真实 API Key 分享给其他人
